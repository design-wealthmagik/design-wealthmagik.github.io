Design Principles

หลักและกฎสำหรับการออกแบบบน UI รวมตั้งแต่หน้าตา, สี, ขนาดและลักษณะคำที่ใช้ประกอบ UI นั้นๆ


Avoid Jargon

การใช้คำย่อ, คำคิดขึ้นใหม่ หรือคำที่ใช้เฉพาะกลุ่ม ควรเป็นคำที่มั่นใจว่าผู้ใช้ส่วนมากจะรู้จักและเข้าใจคำเหล่านั้นดี หากจำเป็นต้องใช้คำเหล่านั้นจริง ควรมีคำบรรยายเพิ่มเติม เพื่ออธิบายให้ผู้ใช้เข้าใจได้ถูกต้อง และหลีกเลี่ยงการเข้าใจผิด

Clear Affordance

การทำให้ UI สื่อสารกับผู้ใช้งานได้ชัดเจนว่า ต้องใช้งานอย่างไรผ่านการออกแบบหน้าตา UI, การใช้คำพูด หรือใช้สีเข้ามาช่วย เช่น การทำให้รูปโดนตัดที่ขอบจอ เพื่อให้รู้ว่าเลื่อนไปต่อได้ หรือการใช้คำกริยาสำหรับปุ่ม เพื่อสื่อว่าปุ่มนี้ใช้ทำอะไร

Less is More

การมีข้อมูลมากเกินจะทำให้คนอ่านสับสน แต่หากจำเป็นต้องแสดงข้อมูลจริงๆ ควรแบ่งเป็นส่วนๆ ว่าอันไหนผู้ใช้ต้องการก่อนและหลังตามลำดับ แล้วแสดงผลตามลำดับนั้น เพื่อให้ ผู้ใช้ค่อยๆ ได้รับข้อมูลตามลำดับที่ถูกต้อง

Be Trustworthy and Credible

เน้นการแสดงภาพลักษณ์ที่น่าเชื่อถือและดูมั่นคง ผ่านการใช้สีและ User Interface ทีตาม standard สากลและไปในทางเดียวกัน รวมไปถึงตัวกราฟฟิคหรือภาพ illustration ที่มี mood and tone ที่เหมือนกัน

Consistency of Style

navigation, elements, color, typography, และ ชุดข้อมูลที่เหมือนกัน ควรทำงานและมีลักษณะเหมือนกันเพื่อจะช่วยให้ผู้ใช้สามารถใช้งานได้ง่ายขึ้น เนื่องจากไม่ต้องเรียนวิธีการใช้งานใหม่ในทุกครั้งของการใช้งาน

Consistency of Information

การให้ข้อมูลต่างๆ บนหน้า UI จำเป็นต้องให้ข้อมูลที่สอดคล้องกัน เพื่อให้ user เข้าใจได้ง่ายและถูกต้อง เช่น สีของกราฟและ label ต้องตรงกัน, หากทำการลบข้อมูลไปแล้ว กราฟไม่ควรแสดงของที่ลบไปแล้ว
